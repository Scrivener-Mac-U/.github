## Overview

<a href="#">Scrivener</a> is Literature & Latte's professional writing application for Mac, purpose-built for long-form writing projects where the limitations of word processors — treating a manuscript as a single continuous document — become genuine obstacles to productive writing. Novelists, screenwriters, academics, journalists, and nonfiction authors use Scrivener as the environment where entire books are drafted, organized, revised, and prepared for publication because its document management model matches how long-form writing actually works rather than how a word processor assumes it should work.

The <a href="#">Binder</a> is Scrivener's organizational backbone — a hierarchical sidebar where every scene, chapter, section, research document, character sheet, and reference file in the project resides as an individual document. Unlike a word processor's single continuous file, Scrivener treats each scene and chapter as a discrete document that can be independently written, reordered, grouped, split, merged, and tagged without affecting adjacent content. <a href="#">Drag-and-drop reorganization</a> restructures the entire narrative by moving scenes and chapters in the Binder — a chapter moved above another immediately changes the manuscript's order, and multiple documents can be selected and moved together as a group. This structural flexibility makes Scrivener the preferred environment for writers who plan extensively before drafting, but equally serves discovery writers who benefit from being able to reorganize freely after drafting exploratively.

<a href="#">The Corkboard view</a> presents each Binder document as an index card on a virtual corkboard, displaying the title and synopsis of each scene in a spatial arrangement that can be reordered by dragging cards to new positions. Writers use the Corkboard to see the shape of their narrative at a glance — identifying pacing problems, structural imbalances, and missing scenes from the card layout in ways that scrolling through continuous prose doesn't reveal. <a href="#">Outliner view</a> presents the same documents as a hierarchical table with custom columns displaying any combination of synopsis, status, label, word count, and custom metadata — enabling the spreadsheet-like overview that plotters use to track scene-level story information across an entire manuscript.

<a href="#">Split editor</a> displays two documents simultaneously in horizontally or vertically split panes — the current drafting scene in one pane and a reference document, character sheet, or earlier chapter in the other. Writing while referencing research, checking earlier established details, or maintaining consistency with previously drafted scenes all benefit from the immediate visual access that side-by-side editing provides. <a href="#">Composition mode</a> expands the active document to fill the entire screen with a configurable background, hiding all other interface elements for distraction-free drafting sessions where only the current text and a minimal toolbar are visible.

## Key Features

- <a href="#">Binder hierarchical document management</a> organizes every scene chapter and reference as individual documents that reorder without affecting others
- <a href="#">Drag-and-drop structural reorganization</a> moves scenes and chapters freely in the Binder to restructure manuscripts without copy-paste operations
- <a href="#">Corkboard index card view</a> displays scene synopses as draggable cards for visual narrative overview and pacing assessment
- <a href="#">Outliner table view</a> presents documents with custom metadata columns for spreadsheet-style tracking of scene status labels and word counts
- <a href="#">Split editor dual-pane writing</a> shows two documents simultaneously for writing while referencing research or earlier manuscript content
- <a href="#">Composition distraction-free mode</a> expands the editor full-screen hiding all interface elements for focused uninterrupted drafting sessions
- <a href="#">Research folder integration</a> stores PDFs images web pages and notes within the project alongside manuscript documents for in-project reference
- <a href="#">Writing targets and progress</a> tracks session and project word count against configurable daily and manuscript completion goals with visual indicators
- <a href="#">Status and label custom metadata</a> assigns color-coded status and label tags to any document for workflow tracking across the entire manuscript

## Additional Information

Scrivener's staying power among professional authors over more than fifteen years reflects the genuine advantage that its document model provides for long-form work. Writers who have completed novels, dissertations, and long-form journalism in Scrivener describe the Binder as transformative — the ability to see the entire project structure, move scenes without surgical copy-paste operations, and keep all research within the same application as the manuscript removes friction that accumulates significantly over hundreds of thousands of words.

Literature & Latte's development of Scrivener has consistently prioritized the needs of working writers over feature proliferation, resulting in an application that remains genuinely focused on helping authors complete long-form projects rather than adding capabilities that serve peripheral use cases. For Mac writers tackling work of any substantial length, Scrivener provides the structural and organizational support that word processors fundamentally cannot.
